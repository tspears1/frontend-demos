[1] Logical Properties

CSS Properites that use values relative to Flow direction (block or inline) and position (start or end) rather than physical direction (top, left, right, bottom) or size (height, width) of the viewport.

Block

The dimension perpendicular to the flow of text within a line, i.e., the vertical dimension in horizontal writing modes, and the horizontal dimension in vertical writing modes. For standard English text, it is the vertical dimension.

Inline

The dimension parallel to the flow of text within a line, i.e., the horizontal dimension in horizontal writing modes, and the vertical dimension in vertical writing modes. For standard English text, it is the horizontal dimension.

[2] Cascade Layers

The @layer CSS at-rule is used to declare a cascade layer and can also be used to define the order of precedence in case of multiple cascade layers.

Syntax

/* Declare layer with rules. */ @layer layer-name { rules } /* Declare layer without rules. */ @layer layer-name; /* Declare multiple layers at once. */ /* Useful for setting order first before difining rules. */ @layer layer-name, layer-name, layer-name; /* Anonymous Layer */ /* Each anonymous layer is unique, and added to the layer order where it is encountered. */ /* Anonymous layers canâ€™t be referenced from other layer rules for sorting or appending more styles. */ @layer { rules } /* import layer (css) */ @import './theme.css' layer(utilities); /* import layer (scss) */ @use 'sass:meta'; @layer buttons { @include meta.load-css('default-button'); /* _default-button.scss */ }

Stacking Order

Layers are stacked in the order they first appear in the stylesheet. The first layer encountered sits at the bottom (least powerful), and the last layer at the top (most powerful). But then, above that, un-layered styles have the highest priority.

@layer layer-1 { a { color: red; } } @layer layer-2 { a { color: orange; } } @layer layer-3 { a { color: yellow; } } /* un-layered */ a { color: green; }
  1. un-layered styles (most powerful)
  2. layer-3
  3. layer-2
  4. layer-1 (least powerful)

Important Layers

Any important styles are applied in a reverse order.

@layer layer-1 { a { color: red !important; } } @layer layer-2 { a { color: orange !important; } } @layer layer-3 { a { color: yellow !important; } } /* un-layered */ a { color: green !important; }
  1. !important layer-1 (most powerful)
  2. !important layer-2
  3. !important layer-3
  4. !important un-layered styles
  5. normal un-layered styles
  6. normal layer-3
  7. normal layer-2
  8. normal layer-1 (least powerful)

Grouped Layers

Layers can also be grouped, allowing us to do more complicated sorting of top-level and nested layers.

@layer layer-1 { a { color: red; } } @layer layer-2 { a { color: orange; } } @layer layer-3 { @layer sub-layer-1 { a { color: yellow; } } @layer sub-layer-2 { a { color: green; } } /* un-nested */ a { color: blue; } } /* un-layered */ a { color: indigo; }
  1. un-layered styles (most powerful)
  2. layer-3
    1. layer-3 un-nested
    2. layer-3 sub-layer-2
    3. layer-3 sub-layer-1
  3. layer-2
  4. layer-1 (least powerful)
/* We can assign to a grouped layer with the following syntax: */ @layer components.stats-card { a { color: yellow; } }

Why use Cascade Layers?

Cascade layers can help us solve several different problems including specificity complexity, organization and greater access to move around the cascade.

@layer framework { .overly#powerful .framework.widget { color: maroon; } } /* Despire the overly specific selector above, this will still win. */ @layer site { .widget { color: rebeccapurple; } }

[3] "S.O.B." Pseudo Class Functions

"S.O.B." (State of Being) Pseudo Class Functions are another set of tools that help us move around the cascade.

They can manipulate specificity without the need for a new selector or allow us to make conditional choices based on the state of an element.

:not()

The :not() pseudo-class function takes a selector list as an argument, and returns true if the element does not match any of the selectors in the list.

/* Selects all elements that are not .fancy elements */ :not(.fancy) { color: coral; } /* Selects all paragraph elements that are not .fancy elements or .super > .cool elements */ p:not(.fancy, .super > .cool) { color: coral; } /* It can also be used to negate pseudo-classes. */ :not(:hover) { color: coral; }

Be very careful not to overuse. This pseudo-class can increase the specificity of a rule. For example, #foo:not(#bar) will match the same element as the simpler #foo, but has the higher specificity of two id selectors.

:where()

The :where() pseudo-class function takes a selector list as an argument, and returns true if the element matches any of the selectors in the list.

It is similar to :is() but always has 0 specificity. This can be very useful for defaults, resets and humbling an elements ego.

ol { color: darkblue; } /* Despite the order of the cascade, the ol will be darkblue because of the zero specificity of the :where() function. */ :where(ol, ul) { color: pink; }
  1. Ordered Item 1
  2. Ordered Item 2
    • Unordered Item 2.1
    • Unordered Item 2.2

It can also be used with other pseudo-class functions. This becomes particularly handy when used with :not(), allowing you to check for the existance of a selector while still maintaing zero specificity so it's easy to override later.

/* Selects all elements that are not .fancy elements */ :where(:not(.fancy)) { color: coral; }

:is()

The :is() pseudo-class function takes a selector list as an argument, and returns true if the element matches any of the selectors in the list.

It is similar to :where() but has the specificity of the most specific selector in the list.

Both pseudo-class functions can be used to group selectors together, making it easier to read and maintain.

/* 3-deep (or more) unordered lists use a square */ ol ol ul, ol ul ul, ol menu ul, ol dir ul, ol ol menu, ol ul menu, ol menu menu, ol dir menu, ol ol dir, ol ul dir, ol menu dir, ol dir dir, ul ol ul, ul ul ul, ul menu ul, ul dir ul, ul ol menu, ul ul menu, ul menu menu, ul dir menu, ul ol dir, ul ul dir, ul menu dir, ul dir dir, menu ol ul, menu ul ul, menu menu ul, menu dir ul, menu ol menu, menu ul menu, menu menu menu, menu dir menu, menu ol dir, menu ul dir, menu menu dir, menu dir dir, dir ol ul, dir ul ul, dir menu ul, dir dir ul, dir ol menu, dir ul menu, dir menu menu, dir dir menu, dir ol dir, dir ul dir, dir menu dir, dir dir dir { list-style-type: square; } /* Using :is() - 3-deep (or more) unordered lists use a square */ :is(ol, ul, menu, dir) :is(ol, ul, menu, dir) :is(ul, menu, dir) { list-style-type: square; }

:has()

The :has() pseudo-class function takes a selector list as an argument, and returns true if the element has an ancestor element that matches any of the selectors in the list.

It is similar to :where() but only matches if the element has an ancestor that matches the selector.

It can be used to style an element based on the existance of a selector in the ancestor chain.

NOTE: This is finally supported in all major browsers.

.fancy-box { background: lightblue; } .fancy-box:has(blockquote) { background: lightgreen; }

This fancy box will be lightblue.

This fancy box will be lightgreen.

Blockquote

:has() can also be used to leverage input values to create change of state in the component, without the need for extra javascript.

In the Logical Properties demo earlier we used the checkbox to change the display state of the helper flags:

.section { --display-flags: none; &:has(input:checked) { --display-flags: inline-block; } .flag { display: var(--display-flags); } }

[4] Advanced Input Media Queries

Media Queries were originally designed to allow us to conditionally set CSS Properties based on the viewport size.

However, gone are the days when we can reliably identify a mobile or tablet device based on screen size alone.

Recently many of the features from Media Queries Version 5 have become very popular:

  • prefers-color-scheme
  • prefers-contrast
  • prefers-reduced-motion

However in Version 4, some oft overlooked features were released that can also help us with accessibility concerns, specifically with how the user interacts with the page:

  • hover
  • pointer
  • any-hover
  • any-pointer

hover

Does the primary input mechanism allow the user to hover over elements?

/* when hover is supported by the primary input */ @media (hover: hover) { a:hover { color: white; background: black; } } /* The primary input mechanism cannot hover at all or cannot conveniently hover */ /* (e.g., many mobile devices emulate hovering when the user performs an inconvenient long tap), */ /* or there is no primary pointing input mechanism. */ @media (hover: none) { a { color: white; background: black; } }

pointer

Is the primary input mechanism a pointing device, and if so, how accurate is it?

/* The primary input mechanism includes an accurate pointing device, such as a mouse. */ @media (pointer: fine) { input[type="checkbox"] { block-size: 15px; border-inline-size: 1px; border-color: blue; inline-size: 15px; } } /* The primary input mechanism includes a pointing device of limited accuracy, such as a finger on a touchscreen. */ @media (pointer: coarse) { input[type="checkbox"] { block-size: 30px; border-inline-size: 2px; border-color: red; inline-size: 30px; } } /* The primary input mechanism does not include a pointing device. */ @media (pointer: none) { input[type="checkbox"] { display: none; } }

any-hover

Does any available input mechanism allow the user to hover over elements?

/* One or more available input mechanisms can conveniently hover over elements. */ /* Such as an iPad using a bluetooth mouse. */ @media (any-hover: hover) { a:hover { color: white; background: black; } } /* None of the available input mechanism(s) can hover conveniently, or there is no pointing input mechanism. */ @media (any-hover: none) { a { color: white; background: black; } }

any-pointer

Is any available input mechanism a pointing device, and if so, how accurate is it?

* At least one input mechanism includes an accurate pointing device. */ @media (any-pointer: fine) { input[type="checkbox"] { block-size: 15px; border-inline-size: 1px; border-color: blue; inline-size: 15px; } } /* At least one input mechanism includes a pointing device of limited accuracy. */ @media (any-pointer: coarse) { input[type="checkbox"] { block-size: 30px; border-inline-size: 2px; border-color: red; inline-size: 30px; } } /* No pointing device is available. */ @media (any-pointer: none) { input[type="checkbox"] { display: none; } }

Usage

Just like the other Media Queries, these can be be combined and used together:

/* The primary input mechanism includes an accurate pointing device, such as a mouse. */ /* AND One or more available input mechanisms can conveniently hover over elements. */ @media (pointer: fine) and (any-hover: hover) { ... }

For quick use, SCSS mixins have been added to the new Component Library:

/* === Hover @media query mixin ============================================ */ /* $any (boolean): True will query 'any-hover', false will query 'hover'. Default is true. */ /* $value (string): The value to query. Options: 'hover' or 'none'. Default is 'hover'. */ @mixin hover($any: true, $value: hover) { @if $any { @media (any-hover: $value) { &:hover { @content; } } } @else { @media (hover: $value) { &:hover { @content; } } } } /* Usage */ .card { @include hover() { background: var(--yellow-5); } }

[5] HSL Color Format

Traditionally we've used hexadecimal format or the rgb() color function.

:root { /* Hexadecimal */ /* The first two digits represent the red channel, the next two the green, and the last two the blue. */ /* In hexadecimal notation, each digit can be 0-9 or A-F. */ /* '00' represents the least intensity of that channel, 'FF' the most. */ --red: #ff0000; --green: #00ff00; --blue: #0000ff; /* RGB */ /* The first number represents the red channel, the second the green, and the last the blue. */ /* In decimal notation, each number can be 0-255. */ /* '0' represents the least intensity of that channel, '255' the most. */ --red: rgb(255 0 0); --green: rgb(0 255 0); --blue: rgb(0 0 255); }

These formats are great for computers, but not so great for humans.

It's hard to read and understand what the color is, and it's hard to manipulate the color.

:root { /* Primary color palette */ --color-primary-50: #f2e5ff; /* Hexadecimal */ rgb(242 229 255); /* rgb() */ --color-primary-100: #e6ccff; /* Hexadecimal */ rgb(230 204 255); /* rgb() */ --color-primary-200: #cc99ff; /* Hexadecimal */ rgb(204 153 255); /* rgb() */ --color-primary-300: #b366ff; /* Hexadecimal */ rgb(179 102 255); /* rgb() */ --color-primary-400: #9933ff; /* Hexadecimal */ rgb(153 51 255); /* rgb() */ --color-primary-500: #8000ff; /* Hexadecimal */ rgb(128 0 255); /* rgb() */ --color-primary-600: #6600cc; /* Hexadecimal */ rgb(102 0 204); /* rgb() */ --color-primary-700: #4d0099; /* Hexadecimal */ rgb(77 0 153); /* rgb() */ --color-primary-800: #330066; /* Hexadecimal */ rgb(51 0 102); /* rgb() */ --color-primary-900: #1a0033; /* Hexadecimal */ rgb(26 0 51); /* rgb() */ --color-primary-950: #0d001a; /* Hexadecimal */ rgb(13 0 26); /* rgb() */ }

A Note on Syntax

Traditionally when writing rgb() or rgba() colors, we've used commas between the values.

However, the CSS Color Module Level 4 spec allows us to use spaces instead, with a slash to separate the alpha channel.

It also no longer requires rgba() to have an alpha channel.

If no alpha channel is provided, it will default to 1.

:root { /* Traditional */ --red: rgb(255, 0, 0); --red: rgba(255, 0, 0, 1); /* Modern */ --red: rgb(255 0 0); --red: rgb(255 0 0 / 1); }

This new syntax also applies to hsl() and hsla() colors.

hsl()

The hsl() color format is another way of expressing color on the sRGB color space. It a great way to work with color. It's easy to read and understand, and allows us to easily manipulate the color.

:root { /* HSL */ /* The first number represents the hue, the second the saturation, and the last the lightness. */ /* The hue is represented as an angle of the color circle, from 0 to 360 degrees. Somtimes the number is followed by 'deg'. */ /* The saturation and lightness are represented as percentages, from 0% to 100%. */ --red: hsl(0 100% 50%); --green: hsl(120 100% 50%); --blue: hsl(240 100% 50%); }

It's also a great way to work with color in CSS, because it allows us to easily manipulate the color using CSS Custom Properties.

:root { /* Primary color palette */ --color-primary-50: hsl(270 100% 95%); --color-primary-100: hsl(271 100% 90%); --color-primary-200: hsl(270 100% 80%); --color-primary-300: hsl(270 100% 70%); --color-primary-400: hsl(270 100% 60%); --color-primary-500: hsl(270 100% 50%); --color-primary-600: hsl(270 100% 40%); --color-primary-700: hsl(270 100% 30%); --color-primary-800: hsl(270 100% 20%); --color-primary-900: hsl(271 100% 10%); --color-primary-950: hsl(270 100% 5%); }

Using CSS Custom Properties, we can also store the HSL values to be used with or without an alpha channel.

These stored values become easier to quickly manipulate for animation or other transitional effects.

:root { --color-primary-500: 270 100% 50%; } .box { background: hsl(var(--color-primary-500)); border-color: hsl(var(--color-primary-500) / 0.5); }
